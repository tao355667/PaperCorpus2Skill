# Teaching Semantic HTML Before Framework Components

## Abstract
This study investigates whether semantic HTML instruction improves learners' ability to reason about accessibility and document structure before using component frameworks.

## Introduction
Component frameworks can accelerate interface development, but they may obscure the relationship between document semantics and assistive technologies. A curriculum that foregrounds semantic HTML may reduce this disconnect.

## Related Work
Prior work on web education has highlighted the tension between practical tooling and foundational platform knowledge. Accessibility-oriented pedagogy further argues that semantic structure should be taught before visual composition.

## Method
The instructional intervention requires students to implement forms, navigation, and article layouts with semantic elements before introducing framework abstractions. Reflection prompts ask students to explain how each element contributes to accessibility.

## Experiments
Learner submissions are evaluated using automated accessibility checks and manual inspection. The semantic-first group shows fewer structural errors in navigation and form labeling tasks.

## Discussion
Semantic HTML appears repeatedly as a conceptual thread: it motivates the pedagogical gap, structures the learning sequence, and explains observed accessibility improvements.
