# Measuring Page Responsiveness in Introductory Web Projects

## Abstract
This paper introduces page responsiveness measurement into introductory web development projects.

## Introduction
Performance metrics are often introduced late in web curricula. As a result, students may optimize aesthetics without considering responsiveness or interaction delay.

## Method
Students measure initial rendering, interaction delay, and layout stability after each project milestone. The measurements are linked to code-level causes such as image size and script loading.

## Experiments
Projects with measurement checkpoints show more consistent responsiveness improvements than projects assessed only at submission time.

## Discussion
Page responsiveness is framed as a user-facing goal, operationalized through measurement, and then used to explain concrete code revisions.

## Conclusion
Early measurement helps students connect technical decisions with perceived interface quality.
