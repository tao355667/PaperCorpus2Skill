# Network-Constrained Assignments in Front-End Courses

## Abstract
The paper proposes network-constrained assignments for teaching performance-aware front-end development.

## Introduction
Students often evaluate web interfaces on fast local machines, which hides the cost of heavy bundles and render-blocking resources. Network constraints make performance trade-offs visible.

## Method
Assignments require students to profile pages under throttled network conditions. The course introduces measurement before optimization so that students justify each change with observable evidence.

## Experiments
The study compares projects before and after the network-constrained module. Students reduce unused JavaScript and improve initial rendering metrics.

## Discussion
Network constraints function as both a motivating problem and an evaluation condition. The same concept is introduced as a real-world limitation, translated into profiling tasks, and used to interpret performance gains.

## Conclusion
Constraint-based assignments can make performance reasoning concrete in front-end education.
